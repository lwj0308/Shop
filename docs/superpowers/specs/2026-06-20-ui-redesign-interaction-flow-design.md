## 一、设计决策总结

| 决策项 | 选择 | 说明 |
|--------|------|------|
| UI 风格 | 极简黑白灰 | 黑白灰主色 + 大量留白 + 衬线字体标题 + 极简边框 |
| 登录交互 | 全部改为弹窗 | 所有需登录的操作都弹出模态框登录，登录后继续原操作 |
| 加购后行为 | 轻量 Toast 提示 | 顶部 Toast 提示"已加入购物车"，2秒自动消失 |
| 详情页布局 | 粘性图片 + 长详情 | 左侧粘性图片（滚动固定），右侧长详情 |
| 首页布局 | 大图 Hero + 网格 | 全屏大图 Hero 区 + 商品分类入口 + 精选商品网格 |
| 导航栏风格 | 透明简洁导航 | 顶部透明导航栏（滚动后变白），Logo + 菜单 + 图标 |
| 实施范围 | 分阶段实施 | 阶段1：全局样式+导航+首页；阶段2：详情页；阶段3：购物车/订单/登录 |
| 未登录加购 | 登录后自动加购 | 弹出登录弹窗 → 登录成功后自动执行加购并 Toast 提示 |

### 2.2 字体系统

```
/* 标题字体：衬线字体（显高级） */
--font-heading: 'Playfair Display', 'Noto Serif SC', Georgia, serif;

/* 正文字体：无衬线字体（清晰易读） */
--font-body: 'Inter', 'PingFang SC', 'Microsoft YaHei', sans-serif;

/* 字号规范 */
--font-size-hero: 56px     /* Hero 大标题 */
--font-size-h1: 36px       /* 页面主标题 */
--font-size-h2: 28px       /* 区块标题 */
--font-size-h3: 20px       /* 卡片标题 */
--font-size-body: 15px     /* 正文 */
--font-size-small: 13px    /* 辅助文字 */
--font-size-caption: 12px  /* 标签/说明 */
```

### 2.3 间距与圆角

```
/* 间距系统（8px 基准） */
--space-xs: 4px
--space-sm: 8px
--space-md: 16px
--space-lg: 24px
--space-xl: 40px
--space-2xl: 64px
--space-3xl: 96px          /* 大区块间距 */

/* 圆角 */
--radius-sm: 2px           /* 小圆角：按钮、标签 */
--radius-md: 4px           /* 中圆角：卡片 */
--radius-lg: 8px           /* 大圆角：弹窗 */
--radius-pill: 999px       /* 胶囊：搜索框 */

/* 阴影 - 极淡 */
--shadow-sm: 0 1px 2px rgba(0, 0, 0, 0.04)
--shadow-md: 0 2px 8px rgba(0, 0, 0, 0.06)
--shadow-lg: 0 8px 24px rgba(0, 0, 0, 0.08)
```

### 2.4 设计原则

1. **大量留白**：区块间距至少 64px，内容呼吸感强
2. **极简边框**：用 1px 浅灰线分隔，避免色块背景
3. **衬线标题**：所有大标题用衬线字体，正文用无衬线
4. **黑白为主**：主色近黑，仅价格用香槟金点缀
5. **微动效**：hover 时平滑过渡（0.3s ease），不夸张
6. **图片优先**：大图展示商品，文字辅助

## 三、交互流程文档

### 3.1 全局交互规则

#### 3.1.1 登录态判断流程

```
用户执行操作（加购/购物车/下单/收藏/关注等）
    ↓
判断是否需要登录？
    ├── 否 → 直接执行操作
    └── 是 → 判断当前登录态
                ├── 已登录 → 直接执行操作
                └── 未登录 → 弹出登录弹窗
                                ↓
                            用户登录
                                ├── 登录成功 → 自动执行原操作 + Toast 提示
                                └── 关闭弹窗/登录失败 → 取消操作，留在当前页
```

**关键点：**
- 登录弹窗是模态框（Modal），不是页面跳转
- 登录成功后，系统自动执行用户原本想做的操作（如加购）
- 登录弹窗支持切换到注册 Tab
- 登录弹窗需记录"待执行操作"（pending action），登录成功后执行

#### 3.1.2 Toast 提示规范

| 场景 | 类型 | 文案 | 时长 |
|------|------|------|------|
| 加入购物车成功 | success | 已加入购物车 | 2s |
| 立即购买跳转 | 无 | 直接跳转 | - |
| 收藏成功 | success | 已收藏 | 2s |
| 关注店铺成功 | success | 已关注 | 2s |
| 操作失败 | error | 具体错误信息 | 3s |
| 需要选择规格 | warning | 请选择商品规格 | 2s |

### 3.2 核心页面交互流程

#### 3.2.1 首页交互流程

```
首页加载
    ↓
展示：透明导航栏 + 全屏 Hero 大图 + 分类入口 + 精选商品网格
    ↓
用户操作：
    ├── 点击导航菜单 → 跳转对应页面
    ├── 点击搜索图标 → 展开搜索框（下拉式）
    │       ├── 输入关键词 → 回车/点击搜索 → 跳转搜索页
    │       └── 点击热搜词 → 直接搜索
    ├── 点击 Hero 区 CTA 按钮 → 跳转分类页或精选商品页
    ├── 点击分类入口 → 跳转分类页
    ├── 点击商品卡片 → 跳转商品详情页
    ├── 点击账户图标 →
    │       ├── 已登录 → 跳转个人中心
    │       └── 未登录 → 弹出登录弹窗
    └── 点击购物车图标 →
            ├── 已登录 → 跳转购物车页
            └── 未登录 → 弹出登录弹窗 → 登录后跳转购物车页
```

#### 3.2.2 商品详情页交互流程

```
进入详情页
    ↓
展示：粘性左侧图片（滚动固定） + 右侧长详情
    ↓
用户操作：
    ├── 点击缩略图 → 切换主图
    ├── 选择 SKU 规格 →
    │       ├── 单 SKU → 自动选中，按钮可点击
    │       └── 多 SKU → 默认选第一个，用户可切换
    ├── 修改数量 → 数量选择器
    ├── 点击"加入购物车" →
    │       ├── 未选 SKU → Toast 警告"请选择规格"
    │       ├── 已选 SKU + 未登录 → 弹出登录弹窗
    │       │       ├── 登录成功 → 自动加购 + Toast"已加入购物车"
    │       │       └── 取消 → 留在详情页
    │       └── 已选 SKU + 已登录 → 直接加购 + Toast"已加入购物车"
    ├── 点击"立即购买" →
    │       ├── 未选 SKU → Toast 警告"请选择规格"
    │       ├── 已选 SKU + 未登录 → 弹出登录弹窗 → 登录后跳转确认订单页
    │       └── 已选 SKU + 已登录 → 跳转确认订单页
    ├── 点击"收藏" →
    │       ├── 未登录 → 弹出登录弹窗 → 登录后执行收藏
    │       └── 已登录 → 执行收藏 + Toast"已收藏"
    └── 滚动页面 → 左侧图片固定，右侧详情滚动
```

#### 3.2.3 购物车交互流程

```
进入购物车页（需登录）
    ↓
未登录 → 弹出登录弹窗 → 登录后跳转购物车页
    ↓
已登录 → 加载购物车列表
    ↓
用户操作：
    ├── 勾选/取消勾选商品 → 实时更新选中状态和小计
    ├── 修改数量 → 防抖 500ms 后更新服务器
    ├── 删除商品 → 确认弹窗 → 删除 + Toast"已删除"
    ├── 全选/取消全选 → 批量更新
    ├── 点击"去结算" →
    │       ├── 无选中商品 → 按钮禁用
    │       └── 有选中商品 → 跳转确认订单页
    └── 继续购物 → 返回首页
```

#### 3.2.4 搜索交互流程

```
用户点击搜索图标/搜索框
    ↓
展开搜索下拉框（含搜索输入框 + 热搜词 + 搜索历史）
    ↓
用户操作：
    ├── 输入关键词 → 回车/点击搜索按钮 → 跳转搜索结果页
    ├── 点击热搜词 → 直接搜索该词
    ├── 点击历史词 → 直接搜索该词
    └── 点击清除历史 → 清空 localStorage 中的历史记录
    ↓
搜索结果页：
    ├── 展示结果统计 + 排序栏 + 价格筛选
    ├── 点击商品卡片 → 跳转详情页
    └── 无结果 → 展示空状态 + 推荐商品
```

#### 3.2.5 下单流程

```
购物车点击"去结算" 或 详情页点击"立即购买"
    ↓
进入确认订单页（需登录）
    ↓
展示：收货地址 + 商品清单 + 优惠 + 支付方式 + 备注
    ↓
用户操作：
    ├── 选择/新增收货地址
    ├── 选择支付方式
    ├── 填写备注
    └── 点击"提交订单" →
            ├── 创建订单成功 → 跳转支付页
            └── 创建失败 → Toast 错误信息
    ↓
支付页 → 支付成功 → 跳转支付结果页
```

#### 3.3.2 待执行操作（Pending Action）机制

```typescript
// 全局存储待执行操作
interface PendingAction {
  description: string      // 操作描述（用于提示）
  execute: () => Promise<void>  // 执行函数
}

// 流程：
// 1. 用户点击需登录的操作（如加购）
// 2. 检测未登录 → 将操作存入 pendingAction
// 3. 弹出登录弹窗，提示"登录后将继续您的操作"
// 4. 登录成功 → 执行 pendingAction.execute() → 清空 pendingAction
// 5. 关闭弹窗 → 清空 pendingAction
```

### 3.4 路由守卫调整

**原逻辑：** 需要登录的页面（cart/order等）未登录时跳转登录页
**新逻辑：**
- 购物车页、订单页等仍保留路由守卫，但未登录时弹出登录弹窗（而非跳转登录页）
- 登录成功后跳转到目标页
- 登录页（/login）保留，但主要用于直接访问场景

```typescript
// stores/auth.ts（新增/改造）
interface AuthState {
  showAuthModal: boolean       // 是否显示登录弹窗
  pendingAction: PendingAction | null  // 待执行操作
}

// 方法：
- openAuthModal(action?: PendingAction)  // 打开弹窗
- closeAuthModal()                       // 关闭弹窗
- executePendingAction()                 // 执行待执行操作
```

### 5.2 路由守卫调整

```typescript
// router/index.ts
router.beforeEach((to, from, next) => {
  if (to.meta.requiresAuth && !isAuthenticated()) {
    // 不再跳转登录页，而是打开登录弹窗
    // 但需要记录目标路由，登录后跳转
    authStore.openAuthModal({
      description: '登录后访问该页面',
      execute: async () => {
        router.push(to.fullPath)
      }
    })
    next(false)  // 取消导航
    return
  }
  next()
})
```
