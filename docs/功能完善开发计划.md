## 三、开发计划与任务分解

### Phase 1：打通用户端核心闭环（P0）

**目标**：让用户能够完成"下单 → 支付 → 查看订单"的完整流程。

#### Task 1.1：订单确认页 ConfirmView.vue
- 调用购物车 API 获取选中的商品
- 调用用户地址 API 获取收货地址列表
- 选择/切换收货地址
- 显示商品清单和金额汇总
- 调用订单创建 API 提交订单
- 下单成功后跳转到支付页

#### Task 1.2：订单列表页 ListView.vue
- 调用订单列表 API（按状态分页查询）
- Tab 切换：全部/待付款/待发货/待收货/已完成/已取消
- 订单卡片展示：商品信息、金额、状态
- 操作按钮：去支付/取消订单/确认收货/查看详情

#### Task 1.3：订单详情页 DetailView.vue
- 调用订单详情 API
- 展示订单状态进度条
- 展示收货地址、商品清单、金额明细
- 展示物流信息（如有）
- 操作按钮：根据状态显示（去支付/取消/确认收货）

#### Task 1.4：支付结果页 ResultView.vue
- 从路由参数获取订单号
- 调用支付查询 API 获取支付状态
- 展示支付成功/失败结果
- 提供"查看订单"和"继续购物"入口

#### Task 1.5：用户中心页 CenterView.vue
- 调用用户信息 API 展示个人资料
- 订单状态快捷入口（待付款/待发货/待收货/退款等数量）
- 收货地址管理入口
- 收藏商品入口
- 浏览足迹入口
- 退出登录

### Phase 2：重要功能完善（P1）

#### Task 2.1：用户端注册页 RegisterView.vue
- 完整注册表单（手机号、验证码、密码、确认密码）
- 调用发送验证码 API
- 调用注册 API
- 注册成功后跳转登录

#### Task 2.2：商家端仪表盘和数据中心接入真实 API
- 后端补充商家端统计接口（如缺失）
- 前端调用真实 API 替换 mock 数据

#### Task 2.3：管理端仪表盘添加图表
- 引入 ECharts 图表库
- 销售趋势折线图
- 订单状态分布饼图
- 商品销量柱状图

### Phase 3：TODO 清理（P2）

- 补充后端重置密码接口
- 补充后端角色权限分配接口
- 完善管理端订单发货弹窗
- 补充商家回复评价接口
- 完善 Dashboard 统计接口（替换硬编码）

### Phase 4：新功能扩展（P3，按需进行）

- 商家结算管理

## 四、开发顺序与依赖关系

```
Phase 1 (P0) → Phase 2 (P1) → Phase 3 (P2) → Phase 4 (P3)
```

**Phase 1 内部顺序**：
1. 先做订单确认页（下单入口）
2. 再做支付结果页（支付后跳转）
3. 再做订单列表页和详情页（查看订单）
4. 最后做用户中心（聚合入口）

**关键依赖**：
- 订单确认页依赖购物车 API 和用户地址 API（后端已就绪）
- 支付结果页依赖支付查询 API（后端已就绪）
- 订单列表/详情依赖订单 API（后端已就绪）
- 所有 P0 任务的后端 API 均已就绪，无需后端开发
