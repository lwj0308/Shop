## 一、跨模块共性问题（最高优先级）

### 1. [严重/阻断] 所有微服务信任 X-User-Id Header，可伪造任意用户身份

涉及：shop-user、shop-product、shop-cart、shop-order、shop-payment、shop-merchant

每个微服务的 `UserInfoInterceptor` 都直接读取 `X-User-Id` 请求头并调用 `StpUtil.login(userId)`。一旦绕过 Gateway 直连服务端口，攻击者加 `X-User-Id: 1` 即可以任意用户身份操作。

### 2. [严重/阻断] 所有内部接口 /inner/** 完全无鉴权

涉及：shop-user、shop-product、shop-order、shop-merchant

`/inner/**` 在 SaTokenConfig 白名单完全放行，无 IP 白名单、无内部签名。可被任意调用造成资损。

### 3. [严重/阻断] 管理端接口普遍缺少角色校验

涉及：shop-user、shop-product、shop-order、shop-merchant

`/admin/**` 接口只校验登录态，普通用户可调用管理接口。

### 4. [必须修复] ThreadLocal 用户身份串号（shop-common）

`SecurityUtils.isLogin()` 用 `UserContext.isLogin() || StpUtil.isLogin()`，ThreadLocal 残留导致越权。

### 5. [必须修复] 数据库默认凭据弱口令

多个模块 `${MYSQL_PASSWORD:root}` 默认 root/root。
